### 模块一：阵列与物理声学环境 (Array & Acoustic Environment)

该模块定义了声呐硬件的基础物理属性和声场的环境参数。如果这里设置错误，会导致后续所有空间谱（CBF/DCV）发生严重畸变。

| **参数名**   | **代码变量** | **默认值** | **参数详解**                                                 | **调优指南 (怎么调，调后效果)**                              |
| ------------ | ------------ | ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **采样率**   | `fs`         | 5000.0     | ADC 模数转换的采样频率 (Hz)。决定了系统能处理的最高频率 (奈奎斯特极限为 fs/2)。 | **不可随意调**。必须与输入的 `.raw` 数据文件的实际采样率严格保持一致，否则所有频率轴完全错乱。 |
| **阵元数量** | `M`          | 512        | 均匀线阵 (ULA) 的传感器个数。决定了阵列的物理孔径和空间分辨率。 | **不可随意调**。必须与实际硬件阵列一致。数量越大，波束越窄，空间分辨力越强。 |
| **阵元间距** | `d`          | 1.2        | 相邻两个水听器之间的物理距离 (米)。                          | **严格匹配硬件**。若随意改大，会导致高频段出现严重的“空间混叠 (栅瓣)”，即一个目标在多个角度出现重影；改小则波束展宽，分辨率下降。 |
| **环境声速** | `c`          | 1500.0     | 水下声波传播速度 (m/s)。受温度、盐度、深度影响。             | **依据海试环境微调** (通常在 1450~1550 之间)。若声速设置与实际不符，会导致波束导向矢量的相位计算偏差，进而使测向角度发生偏移。 |
| **聚焦半径** | `r_scan`     | 9000.0     | 近场/球面波波束形成的扫描聚焦距离 (米)。                     | 对于远场目标(平面波假设)，设为极大的值即可；若目标在近场，调至预估距离可极大提高方位谱的锐度。 |
| **时间步进** | `timeStep`   | 3.0        | 界面瀑布图每次刷新的物理时间间隔 (秒)。                      | **调小**：瀑布图刷新变快，航迹更细腻，但计算量激增；**调大**：宏观上看长航迹更省资源，但容易漏掉瞬态特征。 |

### 模块二：目标特征频段划分 (Frequency Band & Spectrum Estimation)

该模块决定了我们关注哪些频率段的能量。正确的频段划分是滤除海洋背景噪声、锁定目标舰船特征的关键。

| **参数名**        | **代码变量**   | **默认值** | **参数详解**                                                 | **调优指南 (怎么调，调后效果)**                              |
| ----------------- | -------------- | ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **LOFAR 下/上限** | `lofarMin/Max` | 80~250     | LOFAR (低频分析记录) 的分析频带 (Hz)。主要用于提取柴油机、发电机等机械振动的离散明线谱。 | **依据目标类型调**。若找大型货轮，建议调低(如 10~100Hz)；若找高速快艇，适当上移。范围越窄，Tab 3 瀑布图看细节越清晰。 |
| **DEMON 下/上限** | `demonMin/Max` | 350~2000   | DEMON (包络调制解调) 截获宽带噪声的频段。利用螺旋桨空化产生的宽带高频噪声来反推轴频。 | **避开强干扰带**。必须设置在目标宽带辐射噪声强、且没有明显机械线谱干扰的高频区。若该频段信噪比低，DEMON 轴频图(绿线)将没有明显峰值。 |
| **短窗快拍数**    | `nfftR`        | 15000      | 每次处理新读入的数据点数。                                   | **调大**：单帧包含信息更多，抗噪好；**调小**：动态响应快，但容易受瞬态噪声干扰。 |
| **长窗分析点数**  | `nfftWin`      | 30000      | 滑动窗口的总长度及 FFT 点数。决定了最终的频率分辨率 `df = fs / nfftWin`。 | **调大 (如 60000)**：频率分辨率极高(0.08Hz)，能区分靠得很近的线谱，但抹平了快速跳变的特征；**调小**：频域变模糊。 |

### 模块三：实时 LOFAR 线谱提取 (Real-time LOFAR CFAR Extraction)

控制 Tab 1 实时界面中，“红色的线谱拾取图”是否灵敏。

| **参数名**       | **代码变量**         | **默认值** | **参数详解**                                                 | **调优指南 (怎么调，调后效果)**                              |
| ---------------- | -------------------- | ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **背景中值窗宽** | `lofarBgMedWindow`   | 150        | 中值滤波器在频域滑动的窗口宽度 (点数)。用于估计平滑的背景底噪。 | **调大 (如 300)**：背景底噪估计更平滑，容易凸显密集的弱线谱；**调小 (如 50)**：背景估计紧贴原始谱，宽大的强线谱可能会被当成背景给错误地滤除。 |
| **SNR 阈值乘数** | `lofarSnrThreshMult` | 2.5        | 决定了高出背景多少倍的峰才算线谱。公式：`Thresh = Mean + Mult * Std`。 | **调大 (如 4.0)**：系统极其“洁癖”，只提取极其尖锐的高信噪比主线谱，误报极低；**调小 (如 1.0)**：极度敏感，红线图会密密麻麻提取出大量海洋噪点。 |
| **寻峰最小间距** | `lofarPeakMinDist`   | 30         | 频域上两个合法线谱之间必须间隔的最小点数。 (对应频率间距 = `30 * fs / nfftWin = 5Hz`) | **调大 (如 100)**：强行合并靠得近的双线谱；**调小 (如 5)**：一根粗大的线谱（由于FFT能量泄露）会被错误地识别为 3~4 根挨在一起的细线谱，导致冗余。 |

### 模块四：DEMON 包络数字滤波 (DEMON Envelope FIR Filtering)

控制 Tab 1 绿色的 DEMON 包络谱（提取螺旋桨转速）的抗干扰能力。

| **参数名**         | **代码变量** | **默认值** | **参数详解**                                                 | **调优指南 (怎么调，调后效果)**                              |
| ------------------ | ------------ | ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **FIR 滤波器阶数** | `firOrder`   | 64         | 提取宽带噪声包络时所用低通滤波器的阶数。                     | **调大 (如 128)**：滤波器的频域截止特性更陡峭，包络更干净，但带来一定的群延迟和计算开销。 |
| **归一化截止频率** | `firCutoff`  | 0.1        | 低通滤波器的截止点（相对于奈奎斯特频率 `fs/2`）。`0.1` 代表截止频率为 `0.1 * 2500 = 250Hz`。 | **关键调优点**：螺旋桨轴频和叶频通常在 1~50Hz 内。如果将 Cutoff 设置得过大（如 0.5），会有大量高频包络噪声混入绿色的 DEMON 谱中；设为 0.1 刚好卡住有用的低频包络带。 |

### 模块五：TPSW 与 DP 轨迹寻优 (TPSW & DP Track Optimization)

极其硬核的离线后处理参数。决定了 Tab 3 阵列中，第二行的背景均衡效果和第三行的动态规划（DP）线谱轨迹是否连续。

| **参数名**         | **代码变量** | **默认值** | **参数详解**                                                 | **调优指南 (怎么调，调后效果)**                              |
| ------------------ | ------------ | ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **TPSW 保护窗**    | `tpswG`      | 45.0       | Two-Pass Split-Window 的外围大窗口，用于估计背景。           | **调大**：背景估计变得更迟钝，平滑效果好。                   |
| **TPSW 排除窗**    | `tpswE`      | 2.0        | 中心排除窗口。计算背景时，要把线谱本身的宽度掏空排除掉，防止线谱拉高了自身的背景估计。 | **依据目标线谱粗细调节**：如果目标的线谱很宽（存在多普勒展宽），必须加大 E，否则 TPSW 均衡后，线谱能量会被“吃掉”，导致第二行图变暗。 |
| **DP 记忆窗长**    | `dpL`        | 5          | 动态规划算法（Viterbi算法变种）在时间轴上回溯前几帧的最大搜索频点范围。 | **调大 (如 15)**：允许目标线谱在相邻时间帧发生剧烈的频率跳变（比如多普勒机动）；**调小 (如 2)**：强行要求线谱必须是笔直连续的，抗噪极好，但不适应高机动目标。 |
| **惩罚因子 Alpha** | `dpAlpha`    | 1.5        | 调节对“节点能量强度”的惩罚力度。                             | **调大 (如 3.0)**：只有能量极其强悍的连续轨迹才会被保留；**调小 (如 0.5)**：允许能量较弱、时断时续的暗弱轨迹被连接起来（Tab 3 最底下的图会显示更多的离散微弱轨迹）。 |
| **惩罚因子 Beta**  | `dpBeta`     | 1.0        | 调节对“频率跳变（轨迹拐弯）”的惩罚力度。                     | **调大 (如 5.0)**：算法极度讨厌弯曲，即使轨迹断掉也不去连接旁边频率偏差大的点；**调小 (如 0.1)**：允许画出极其弯曲甚至折线状的线谱轨迹（适合观察大幅度加减速的目标）。 |
| **偏置因子 Gamma** | `dpGamma`    | 0.1        | 算法分母的平滑项，防止出现除以 0 的奇点崩溃。                | **一般无需修改**。保持极小的值（0.1 ~ 0.5）提供稳定偏置即可。 |



`rl_iter` 代表 **Richardson-Lucy (RL) 反卷积算法的迭代次数**。

在这段声呐信号处理代码中，常规波束形成（CBF）得到的空间谱往往是“模糊”的（波束主瓣较宽，存在旁瓣）。这段代码的核心思想是：利用阵列和系统固有的**点扩散函数（PSF，即理想点目标的系统响应）**，对模糊的常规空间谱（`P_uxuy`）进行**反卷积（Deconvolution）**，从而把模糊的能量重新汇聚成极细的锐角（高分辨谱 `S_uxuy`）。

`rl_iter = 20` 意味着这个反卷积过程要迭代计算 20 次。

### 如何调整 `rl_iter` 来优化方位估计效果？

调整这个参数本质上是在**“极高分辨率”**和**“系统抗噪能力（计算性能）”**之间寻找平衡。以下是具体的调整策略和现象：

#### 1. 调大 `rl_iter`（例如：30、50、甚至 100）

- **现象与优点**：反卷积进行的更彻底。目标在 DCV 空间谱上的波束（主瓣）会变得**非常窄、非常尖锐**。如果你有两个靠得非常近的目标（常规波束下糊成了一团），增加迭代次数可以成功把它们“劈开”成两个独立的峰。
- **副作用（缺点）**：
  - **噪声放大（过拟合）**：RL 算法在迭代过多时，会把背景噪声也当成微弱的信号去“锐化”，导致谱图上出现大量针尖状的**虚假目标（伪影）**。
  - **算力急剧增加**：RL 算法包含密集的矩阵乘法运算，迭代次数与耗时成正比。如果调得太高，你的 `DspWorker` 处理一帧的时间可能超过声呐数据产生的时间（例如 > 3秒），导致系统卡顿或数据积压。

#### 2. 调小 `rl_iter`（例如：5、10、15）

- **现象与优点**：背景噪声会更平滑，不那么容易出现虚假目标点。同时计算速度非常快，极大降低 CPU 的负载。
- **副作用（缺点）**：反卷积不彻底（欠拟合）。去模糊的效果不明显，DCV 高分辨谱的优势发挥不出来，目标的波束依然比较宽，难以区分密集目标。

### 💡 实战优化建议（结合你刚加入界面的输入框）

当你通过 UI 界面调试时，建议按照以下步骤寻找最优值：

1. **观察底噪与假目标**：如果你在右侧的“DCV (高分辨)”折线图中发现除了真实目标外，背景上到处都是毛刺状的小峰，这说明当前的信噪比较低，且迭代次数过高了，此时应**下调 `rl_iter`（如降至 10~15）**以压制噪声。
2. **观察分离度**：如果你确信前方有两个目标，但它们在空间谱上连在了一起没有分开，此时可以**逐步上调 `rl_iter`（如 30 -> 40 -> 50）**，直到看到主瓣分裂成两个。
3. **结合环境信噪比（SNR）动态调整**：
   - **近距离/强目标/高信噪比环境**：大胆增加迭代次数（30 ~ 60），以获取极致的方位分辨率。
   - **远距离/弱目标/低信噪比环境**：保守设置迭代次数（10 ~ 20），以防丢失真实目标或被虚假噪声误导。
4. **注意性能瓶颈**：调整参数后，看看左侧日志里打印的“处理一帧耗时”。确保这个耗时小于你设置的“时间步进”（如 3.0s），以保证软件能够真正的**实时**运行。
